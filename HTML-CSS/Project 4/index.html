Introduction

To date, the C# programming language is one of the most powerful, rapidly developing and sought-after languages ​​in the IT industry. At present, various applications are written on it: from small desktop programs to large web portals and web services that serve millions of users every day.

C# is a language with C-like syntax and is close in this respect to C++ and Java. Therefore, if you are familiar with one of these languages, it will be easier to master C#.

C# is object-oriented and, in this respect, has learned a lot from Java and C++. For example, C# supports polymorphism, inheritance, operator overloading, static typing. The object-oriented approach allows to solve problems of building large, but at the same time flexible, scalable and extensible applications. And C # continues to actively develop, and with each new version, more and more interesting functionalities appear, such as lambdas, dynamic linking, asynchronous methods, etc.

First programm

It is time to try to write something... We think that it will not surprise anyone if the first application is the standard "Hello, World".

namespace CSharpApplication.HelloWorld { class FirstApplication { static void Main() { System.Console.WriteLine("Привет, C#!\n"); } } }

The namespace keyword creates a namespace in which the user can create their own classes without fear of matching names.

In C #, there are no global variables and functions, so all declarations of variables and functions must be inside a class declaration.

The entry point to the application is the Main function. The function must be declared as static, since in this case it is not required to create a class object to call the function.

Variables

First comes the type of the variable, then its name. The name of a variable can be any arbitrary name that satisfies the following requirements:

  • the name can contain any numbers, letters, and underscore, with the first character in the name must be a letter or underscore;
  • no punctuation marks or spaces in the name;
  • The name cannot be a C # keyword. There are not many such words, and when working in Visual Studio, the development environment highlights the keywords in blue.

For example, let's define the simplest variable:

string name;

After defining the variable, you can assign a value:

string name; name = "Tom";

We can also assign a variable to a value immediately after the definition. This technique is called initialization:

string name = "Tom";

A distinctive feature of variables is that in the program you can repeatedly change their value.

Data types

C# has the following basic data types:

  • bool: stores true or false (logical literals). Represented by the System.Boolean system type;
  • byte: stores an integer from 0 to 255 and takes 1 byte. Represented by the System.Byte system type;
  • sbyte: stores an integer from -128 to 127 and takes 1 byte. Represented by the System.SByte system type;
  • short: stores an integer from -32768 to 32767 and takes 2 bytes. Represented by the System.Int16 system type;
  • ushort: Stores an integer from 0 to 65535 and takes 2 bytes. Represented by the System.UInt16 system type;
  • int: stores an integer from -2147483648 to 2147483647 and takes 4 bytes. Represented by the System.Int32 system type. All default integer literals represent int values;
  • uint: Stores an integer from 0 to 4294967295 and takes 4 bytes. Represented by the System.UInt32 system type;
  • long: stores an integer from -9223372036854775808 to 9223372036854775807 and occupies 8 bytes. Represented by the System.Int64 system type;
  • ulong: stores an integer from 0 to 18446744073709551615 and occupies 8 bytes. Represented by the System.UInt64 system type;
  • char: stores a single character in Unicode and takes 2 bytes. Represented by the System.Char system type. This type corresponds to character literals;
  • string: stores the Unicode character set. Represented by the System.String system type. This type corresponds to character literals;
  • object: can store a value of any data type and occupies 4 bytes on a 32-bit platform and 8 bytes on a 64-bit platform. Represented by the System.Object system type, which is the base type for all other .NET types and classes.

Arithmetic operations

The operation of adding two numbers:

int x = 10; int z = x + 12; // 22

The operation of subtracting two numbers:

int x = 10; int z = x - 6; // 4

The operation of multiplying two numbers:

int x = 10; int z = x * 5; // 50

The operation of dividing two numbers:

int x = 10; int z = x / 5; // 2

The operation of obtaining the remainder of the integer division of two numbers:

double x = 10.0; double z = x % 4.0; // 2

Increment operation:

int x1 = 5; int z1 = ++x1; // z1=6; x1=6 int z2 = x1++; // z2=5; x1=6

The operation of decrement or decrement value by one:

int x1 = 5; int z1 = --x1; // z1=4; x1=4 int z2 = x1--; // z2=5; x1=4

When performing several arithmetic operations at once, the order of their execution should be considered. Priority of operations from highest to lowest:

  1. Increment decrement
  2. Multiplication, division, obtaining the remainder
  3. Addition, Subtraction

To change the order of operations, brackets are used.

Logical operations

The operation of logical addition or logical OR. Returns true if at least one of the operands returns true:

bool x1 = (5 > 6) | (4 < 6); // true bool x2 = (5 > 6) | (4 > 6); // false

The logical multiplication or logical AND. Returns true if both operands are both true:

bool x1 = (5 > 6) & (4 < 6); // false bool x2 = (5 < 6) & (4 < 6); // true

The operation of logical addition. Returns true if at least one of the operands returns true:

bool x1 = (5 > 6) || (4 < 6); // true bool x2 = (5 > 6) || (4 > 6); // false

Logical multiplication operation. Returns true if both operands are both true at the same time:

bool x1 = (5 > 6) && (4 < 6); // false bool x2 = (5 < 6) && (4 < 6); // true

The operation of logical negation. It is performed on one operand and returns true if the operand is false. If the operand is true, the operation returns false:

bool a = true; bool b = !a; // false

The operation is exclusive OR. Returns true if either the first or second operand (but not simultaneously) are true, otherwise returns false:

bool x5 = (5 > 6) ^ (4 < 6); // true bool x6 = (50 > 6) ^ (4 / 2 < 3); // false

Conditional constructions

The if/else construct checks the validity of a condition and, depending on the results of the check, executes certain code:

int num1 = 8; int num2 = 6; if(num1 > num2) Console.WriteLine($"{num1} > {num2}"); else if (num1 < num2) Console.WriteLine($"{num1} < {num2}"); else Console.WriteLine($"{num1} = {num2}");

The switch/case construction is similar to the if/else construction, since it allows you to process several conditions at once:

Console.WriteLine("Press Y or N"); string selection = Console.ReadLine(); switch (selection) { case "Y": Console.WriteLine("You pressed Y"); break; case "N": Console.WriteLine("You pressed N"); break; default: Console.WriteLine("You pressed unknown button"); break; }

The ternary operation has the following syntax: [first operand - condition]?[second operand]:[third operand]. There are three operands here. Depending on the condition, the ternary operation returns the second or third operand: if the condition is true, then the second operand is returned; if the condition is false, then the third. For example:

int x=3; int y=2; Console.WriteLine("Press + or -"); string selection = Console.ReadLine(); int z = selection=="+"? (x+y) : (x-y); Console.WriteLine(z);

Cycles

Cycles are control structures, allowing depending on certain conditions to perform a certain action many times. C# has the following types of cycles:

  • for
  • foreach
  • while
  • do...while

Consider the standard for loop:

for (int i = 0; i < 9; i++) Console.WriteLine(i);

The cycle block will work 9 times until the value of i becomes 9. And each time this value will increase by 1.

In the do loop, the loop code is first executed, and then the condition is checked in the while statement. And while this condition is true, the cycle repeats. For example:

int i = 6; do { Console.WriteLine(i); i--; } while (i > 0);

Here the loop code will work 6 times until i equals zero. But it is important to note that the do loop guarantees at least one-time execution of actions, even if the condition in the while statement is not true.

In contrast to the do loop, the while loop immediately checks the truth of some condition, and if the condition is true, then the loop code is executed:

int i = 6; while (i > 0) { Console.WriteLine(i); i--; }
